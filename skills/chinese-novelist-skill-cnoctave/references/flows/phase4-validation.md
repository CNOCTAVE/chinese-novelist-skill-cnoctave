# 第四阶段：自动校验与修复

**目标**：逐章校验每一章的字数与完成度，不合格章节当场重写修复（每章最多3轮），全程无需用户介入。

> **校验原则**：
> - **串行模式 [A]**：严格【一次只处理一章】，当前章校验与修复全部完成后，才处理下一章（见下方「逐章校验路径」）。
> - **并行模式 [B]/[C]**：在 Phase 3 批末已由主 Agent 将各独立状态文件合并进 `02-写作计划.json`，本阶段对【本批各章】并发校验（见下方「批量校验路径」），不合格章计入下一批重试。

---

## 校验流程

### 1. 初始化校验

读取 `02-写作计划.json`，将项目 `status` 更新为 `"validating"`。

### 2a. 逐章校验路径（串行模式 [A]）

按章节编号从小到大，**一次只处理一章**，对每一章依次执行：

1. **文件存在性**：检查 `filePath` 指定的文件是否存在
2. **字数检查**（必须）：
   ```bash
   python scripts/check_chapter_wordcount.py <章节文件路径>
   ```
   将 `wordCount` 和 `wordCountPass` 写入【当前这一章】的记录
3. **若 `wordCountPass == false` → 当场修复【当前这一章】（最多3轮）**：
   a. 将本章 `status` 设为 `"failed"`，`retryCount` 加 1
   b. 回到第三阶段「逐章创作流程」重写【当前这一章】（写前分析→撰写→润色→收尾，并以独立状态文件记录结果）
   c. 重写完成后重新执行字数检查
   d. 更新 `02-写作计划.json`
   e. 仍不通过且 `retryCount < 3` → 回到 a 再试；达到 3 次仍不通过 → 保留记录、标注但不阻塞，继续下一章
4. **当前章校验/修复全部完成后，再处理下一章**，绝不跨章批量操作

### 2b. 批量校验路径（并行模式 [B]/[C]）

Phase 3 每批子 Agent 返回后，主 Agent 已将该批各章独立状态文件（`状态/第XX章-state.json`）合并进 `02-写作计划.json`。本阶段对该批各章并发校验：

1. **文件存在性**：批量检查本批各章 `filePath` 指定文件是否存在（缺失 = 子 Agent 失败，标记 `failed`）
2. **字数检查**（可并发执行）：
   ```bash
   python scripts/check_chapter_wordcount.py <第N章文件路径>
   python scripts/check_chapter_wordcount.py <第N+1章文件路径>
   # 或使用 --all 一次检查整批
   python scripts/check_chapter_wordcount.py --all <项目文件夹>/
   ```
   将 `wordCount` 和 `wordCountPass` 写入对应章记录
3. **不合格章（wordCountPass=false 或 failed）→ 本批内并发修复**（每章最多3轮）：
   a. 将该章 `status` 设为 `"failed"`，`retryCount` 加 1
   b. 回到第三阶段，为该章**单独派发一个重写子 Agent**（写前分析→撰写→润色→收尾），重写结果写入其独立状态文件
   c. 多个不合格章可【同一条消息】并发派发重写子 Agent
   d. 重写完成后重新执行字数检查并更新 `02-写作计划.json`
   e. 仍不通过且 `retryCount < 3` → 再次并发重写；达到 3 次仍不通过 → 保留记录、标注但不阻塞其他章
4. **本批全部通过或达上限后**，再处理下一批（Phase 3 主循环控制），绝不跨批阻塞。

### 3. 汇总结果

所有章节依次处理完毕：

- **全部通过** → 更新项目 `status` 为 `"completed"`，进入步骤4
- **存在超3次仍不合格章节** → 仍更新项目 `status` 为 `"completed"`，但在报告中标注（见步骤4）

### 4. 完成报告

向用户展示创作完成总结：

```
📊 《[小说名称]》创作完成

总章数：[X] 章
总字数：[X] 字
完成率：[X]%

各章节状态：
✅ 第1章：[标题]（[字数]字）
✅ 第2章：[标题]（[字数]字）
...

项目文件夹：./chinese-novelist/[timestamp]-[小说名称]/
```

如有不合格章节（超过3次重写仍未通过），在报告中标注：
```
⚠️ 第X章：字数不足（[实际字数]/3000），已重试3次
```
