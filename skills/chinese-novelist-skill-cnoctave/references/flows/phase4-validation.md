# 第四阶段：自动校验与修复

**目标**：确保所有章节完成且字数达标，全程无需用户介入。

---

## 校验流程

### 1. 初始化校验

读取 `02-写作计划.json`，将项目 `status` 更新为 `"validating"`。

### 2. 逐章检查

（必须）对每一章执行以下检查：

1. **文件存在性**：检查 `filePath` 指定的文件是否存在
2. **更新 JSON**：将 `wordCount` 和 `wordCountPass` 写入对应章节记录

（可选）对每一章执行以下检查。重要：如果在自动化工具中运行此skill，则跳过这些可选的步骤：
1. **字数检查**：使用脚本检查字数
   ```bash
   python scripts/check_chapter_wordcount.py <章节文件路径>
   ```
2. **更新 JSON**：将 `wordCount` 和 `wordCountPass` 写入对应章节记录

### 3. 汇总结果

- **全部通过**（所有章节文件存在且 `wordCountPass == true`）→ 更新项目 `status` 为 `"completed"`，进入步骤5
- **有不通过** → 进入步骤4

### 4. 自动修复（最多3轮）

对每个不通过的章节：

1. 将 `status` 设为 `"failed"`，`retryCount` 加 1
2. 回到第三阶段对该章节执行逐章创作流程（写前分析→撰写→润色→收尾）
3. 重写完成后重新检查字数
4. 更新 `02-写作计划.json`

循环规则：
- 最多执行 **3轮** 校验-修复循环
- 超过3次仍不合格的章节，保留记录但不阻塞流程
- 所有修复完成后更新项目 `status` 为 `"completed"`

### 5. 完成报告

向用户展示创作完成总结：

```
📊 《[小说名称]》创作完成

总章数：[X] 章
总字数：[X] 字
完成率：[X]%

各章节状态：
✅ 第1章：[标题]（[字数]字）
✅ 第2章：[标题]（[字数]字）
...

项目文件夹：./chinese-novelist/[timestamp]-[小说名称]/
```

如有不合格章节（超过3次重写仍未通过），在报告中标注：
```
⚠️ 第X章：字数不足（[实际字数]/3000），已重试3次
```

---

### 6. 可选增强阶段（校验通过后）

校验通过后，使用 AskUserQuestion 工具让用户选择是否运行可选增强阶段：

Question: 创作已完成！是否需要对小说进行可选增强？
[A] 强化故事矛盾（女频优化方向）→ 优化人物对立、多重压力、两难抉择等
[B] 增强故事代入感（男频优化方向）→ 优化感官描写、情绪绑定、缩小视角等
[C] 两个都运行→ 先强化矛盾，再增强代入感
[D] 都不需要，直接结束
[E] AI 去重优化 → 查重+降重，消除跨章节重复内容

→ 用户选择后，跳转到对应的可选阶段执行。详见 [SKILL.md](../../SKILL.md) 第4.5步说明。
