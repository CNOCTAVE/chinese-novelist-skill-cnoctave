# 第1000阶段：（可选）AI 查重与去重优化

**用户通过 AskUserQuestion 选择运行此阶段后执行**
**目标**：检测并消除跨章节重复内容，降低 AI 查重率

> **背景**：AI 在分章创作时容易产生重复——有的章节会和之前的章节有几百字雷同，包括一字不差的复制和仅变几个字的"伪差异"。本阶段专为此问题设计。

---

## 一、查重检测方法

### 1. 滑动窗口查重

- 以 **50 字**为窗口，从每章开头向结尾滑动，步长 10 字
- 将每章的窗口内容与相邻章（第 N 章 vs 第 N±1 章）的窗口逐一比对
- 标记完全匹配段落（一字不差）
- 标记高相似度段落（变化 ≤3 个字的句子群）

### 2. 跨章节语义查重

- 检查跨 **3 章以上**的重复场景/动作/对话模板
- 检测"伪差异"：同一段叙事被复制后仅改了角色名、时间、地点
  - 示例：第3章"他推开木门，吱呀一声惊动了屋里的人" → 第5章"她推开铁门，哐当一声惊动了屋里的人"→ 语义重复
- 标记所有"结构相同、细节微调"的段落

### 3. 高频句式检测

- 统计 AI 倾向性高频句式：
  - "他/她感到……"
  - "就在这时……"
  - "没错，……"
  - "突然，……"
  - "某种说不清的……"
  - "不由自主地……"
- 标记单章出现 >3 次或全篇出现 > 章节数×2 次的句式

### 4. 重复报告格式

```
## 查重报告

总章节数：[N]
发现重复段落：[N] 处
  - 完全匹配：[N] 处
  - 高相似度（变化≤3字）：[N] 处
  - 语义重复： [N] 处
高频句式超标：[N] 种
去重目标：全部处理完成
```

## 二、去重处理优先级

### 优先级一：删除（对故事无影响的重复）

- 完全复制的段落 → 删除后文重复，仅保留首次出现
- 仅变了角色名的重复场景 → 删除冗余，保留更完整的一版
- 上下章结尾/开头的重复叙事 → 精简为 1~2 句流畅过渡
  - 示例：第2章结尾"他决定明天一早就去警局问个清楚" + 第3章开头"第二天一早，他来到警局" → 合并为"第二天一早，他推开警局的玻璃门"

### 优先级二：改写（需保留信息但换表达）

- **同一人物的重复出场描写**：每章用不同视角/细节刻画，避免"他穿着一件黑色风衣"×N
  - 第1次：客观外貌 → 第2次：通过他人目光 → 第3次：通过动作暗示
- **同一环境的重复描写**：分散在不同章节中侧重不同感官维度
  - 第N次出现：只写"不同"——天气变化、光线变化、季节变化
- **重复的过渡句式**：替换为不同的衔接手法
  - "第二天" → "次日清晨"/"太阳还没升起来的时候"/"闹钟响了三次他才爬起来"
- **相同情节的复述**：角色回忆已发生的事时，不要原样复述原文，用不同的措辞概括

### 优先级三：结构重组（改动较大时）

- 连续 **3 章以上**开头结构雷同 → 重新设计每章首段切入点
- 对话模式高度重复的章节 → 为每场对话赋予不同的潜台词目标
- 多章使用相同"悬念—揭晓"节奏 → 调整部分章节的悬念位置

## 三、去重改写技巧

### 技巧一：同义替换 + 句式重构

用不同的句式和词汇表达相同的信息。

| 原表达 | 去重改写 |
|--------|---------|
| 他感到一阵寒意从背后升起 | 后背像被人贴了一块冰 |
| 她不由自主地颤抖了一下 | 指尖抖了一下，她赶紧把手藏到身后 |
| 就在这时，门突然开了 | 门被推开了，没有任何预兆 |
| 他心中涌起一种说不清的复杂情绪 | 他说不清自己现在是愤怒还是如释重负 |
| 夜幕降临，城市的灯光次第亮起 | 天色暗下来之后，窗户一扇接一扇地亮了 |

### 技巧二：视角切换法

同一事件在不同章节中从不同角色视角叙述，避免全知重复。

- 第 N 章的 A 视角描写 → 第 N+2 章的 B 视角不能重复相同的感官细节
- A 视角的重点（他看到什么）≠ B 视角的重点（他在想什么）

### 技巧三：细节差异化

同一场景每次出现只突出 **1~2 个新细节**，旧细节通过角色行为暗示环境延续。

- 示例：主角的家→第1次写沙发和窗帘的颜色→第2次写窗台上枯萎的植物→第3次写冰箱上贴的便签
- 规则：**不重复描述已写过的物体特征**，除非有新变化

### 技巧四：合并法

- 相邻两章的环境铺垫完全相同 → 合并为一章精炼过渡，另一章直接切入事件
- 同一解释性文字出现在多处 → 只保留首次出现的位置
- 对话中重复确认信息（"你刚才说……？""对，我是说……"）→ 精简为一句

### 技巧五：时间跳跃法

- 如果两章之间的时间线有重叠（同一时间段被写了两次）→ 调整各自的起止时间，消除重叠
- 重叠部分必产生重复，通过错开时间线从根本上杜绝

## 四、操作流程

> **逐章原则**：去重的【检测阶段】需通读全篇以发现跨章节重复（这一步允许读全部章节）；但【改写阶段】严格逐章执行——一次只重写涉及【当前章】的重复段落，改完一章再处理下一章，禁止一次性批量改写多章。

### Step 1：全量查重

1. 读取 `02-写作计划.json`，获取完整章节列表和文件路径
2. 读取所有章节文件内容（仅为检测重复，不在此阶段改写）
3. 执行四项检测方法，生成查重报告
4. 更新项目 `status` 为 `"deduplicating"`

### Step 2：逐章改写（核心循环）

按章节编号从小到大，**一次只处理一章**，对每一章依次执行：

1. 在查重报告中筛选出【涉及当前章】的重复处
2. 仅对该章上下文（前后各 200 字）按优先级处理（删除→改写→重组）
3. 执行修改，**只写回当前这一章**的文件
4. 记录修改类型和位置
5. 当前章改写完成后，再处理下一章

> 若同一处重复涉及两章（如相邻章结尾/开头重复），只在【后出现的那一章】中修改，前一章保持不变，避免跨章批量改写。

> **重要**：每次修改前必须确认该处改写不会破坏：
> - 人物性格一致性
> - 情节逻辑连贯性
> - 悬念钩子的有效性

### Step 3：重写后检查

对每一处修改完成后的章节执行：

1. **字数检查**：`python scripts/check_chapter_wordcount.py <章节文件路径>`
   - 如果删除后字数低于 3000 → 使用 [content-expansion.md](../guides/content-expansion.md) 补充**新**内容
   - 新内容不能引入新的重复
2. **连贯性检查**：修改后的段落与前后文是否自然衔接
3. **无新重复检查**：确保修改没有意外引入新的重复

### Step 4：更新状态与完成报告

1. 更新 `02-写作计划.json` — 将 `status` 设为 `"completed"`，记录去重信息
2. 展示完成总结：

```
📊 《[小说名称]》AI 去重完成

处理重复：[X] 处
- 删除：[X] 处
- 改写：[X] 处
- 结构重组：[X] 处

高频句式优化：[X] 种
受影响章节：[X] 章
```

## 五、避坑指南

### 不要过度删除
- 保留必要的"承接上章"内容（1~2 句足够），让读者能跟上剧情
- 保留角色标志性特征（口头禅、习惯动作），这是角色辨识度

### 不要破坏悬念
- 章节结尾的悬念钩子即使与上一章开头的承接有部分重叠，也不删除
- 优先改开头承接而非删结尾钩子

### 不要引入新问题
- 改写后的措辞不能比原文差（抽象程度、细节密度）
- 删除后必须确保指代清晰（"他"要能明确对应到角色）
- 跨章节引用保持一致（角色名字、地点名字不用改）
