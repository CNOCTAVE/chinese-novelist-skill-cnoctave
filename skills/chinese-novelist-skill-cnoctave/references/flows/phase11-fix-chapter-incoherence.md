# 第11阶段：（可选）（多用于女频）章节内容连贯性修复——解决"内容不连贯、读着跳"的问题

**用户通过 AskUserQuestion 选择运行此阶段后执行**
**目标**：检测并消除"内容不连贯"类问题——情感跳跃、关系进展突兀、情节线索断档、时空场景硬切、视角人称乱跳、行为动机缺失等。让读者感到故事"丝滑衔接、一气呵成"，杜绝"这一章和下章接不上""情绪怎么突然变了""刚才还在A地怎么就到B地了""他为什么突然这样做"等出戏问题。

> **与第10阶段的边界（务必分清）**：
> - 第10阶段修「**矛盾**」——前后**逻辑上不能同时为真**的东西（人设崩了、时间线对不上、情节前后打架）。
> - 本阶段修「**不连贯**」——前后**逻辑上不冲突、但读起来断档、跳脱、缺衔接**的东西（情绪没铺垫就突变、关系一夜之间变了、线索写一半没了、场景硬切没过渡）。
> - 简单判据：**两处能否同真？** 不能同真 → 第10阶段（矛盾）；能同真但读着别扭、缺过渡 → 本阶段（不连贯）。

## 何时运行本阶段（两个时机）

与第10阶段类似，本阶段也在两个时机被触发，均由用户**主动选择**是否运行：

- **时机一：用户改完大纲 / 人物设定之后、开始写作之前（Timing 1）**
  用户在第二阶段「暂停修改」或直接编辑了设定文件后，于后续对话输入「继续写作」恢复创作。此时用户很可能**漏改了连贯性线索**——比如把第 3 章的感情线改成了"决裂"，却忘了第 4 章仍按"热恋"的连贯基调写，导致章间情感断裂；或改了时间线却没补过渡。本阶段在恢复写作前提供检测，先把连贯性断层掐灭。
- **时机二：skill 已写完小说内容（Timing 2）**
  自动校验（第 4 阶段）通过后，在「第 4.5 步可选增强阶段」中作为选项提供。用于修复分章创作过程中**自身产生**的不连贯（情感跳跃、关系推进突兀、线索断档、硬切等）。

> 两种时机共用同一套检测 / 呈现 / 修复逻辑，区别仅在**可用材料**（同第10阶段）。本阶段以**「前文已建立的连贯基线」**为锚点——后文需与这条基线平滑衔接。

## 0. 运行前准备（确认可用材料）

1. 读取 `02-写作计划.json`，确认 `projectPath`、`status`、章节列表与文件路径。
2. 读取 `00-人物档案.json`（人物设定基准）与 `01-大纲.json`（情节走向基准，含每章摘要 `summary`，是连贯性参考）。
3. 根据 `status` 判断可用材料：
   - 已写章节 → 读取这些章节 `.md`，按下方六个维度 + 章间扫描做连贯性检测。
   - 仅有人物档案 + 大纲（Timing 1 尚未开写）→ 聚焦「大纲层连贯性」与各章 `summary` ↔ `summary` 的衔接。
4. 更新 `02-写作计划.json` 的 `status` 为 `"coherence_fixing"`（仅当存在已写章节需要修改时）。

## 一、连贯性检测（仅检测，不修改）

按以下六个维度逐项比对，记录每一处不连贯的**位置、断层表现、所属维度、衔接缺口**。

### 维度一：情感连贯（女频核心，最易"跳"）

| 不连贯类型 | 说明 | 示例 |
|------------|------|------|
| 情绪跳跃 | 情绪无铺垫地突变，缺触发→积累→爆发→余韵 | 上一段还在甜蜜傻笑，下一段毫无征兆地"崩溃大哭、心如死灰" |
| 情绪基调断裂 | 前后情绪基调不兼容、无转折交代 | 第3章结尾"彻底放下了他"，第4章开篇"满心欢喜等他归来"且中间无事件触发反转 |
| 情感逻辑断裂 | 情感反应与人物处境 / 性格不符，缺内心依据 | 刚得知挚爱死讯，下一句却"轻松地哼着歌逛街"，无麻木 / 逃避等心理过渡 |
| 余韵缺失 | 强情绪场景后直接切下一个事件，情绪没落地 | 痛哭一场后，紧接着一句"第二天她准时上班"，中间无平复 / 缓冲 |

### 维度二：人物关系 / CP 连贯（女频核心）

| 不连贯类型 | 说明 | 示例 |
|------------|------|------|
| 关系进展突兀 | 关系亲疏程度前后不符，缺阶段过渡 | 第5章还"形同陌路、互相防备"，第6章已"亲密无间、互诉衷肠"，中间无破冰事件 |
| 关系状态前后错位 | 关系状态前后矛盾却非逻辑冲突（更像忘了衔接） | 第3章"两人已和好如初"，第7章又写"两人至今冷战未说话"——非矛盾而是衔接断层 |
| CP 甜蜜 / 虐感断档 | 情感浓度忽高忽低，无节奏依据 | 连续三章高浓度撒糖后，第4章突然零互动零张力，无任何剧情理由 |
| 称谓 / 互动方式突变 | 对彼此的称呼、相处模式前后不统一 | 前文"陆哥哥""晚晚"腻称，后文突然全名相称且无决裂交代 |

### 维度三：情节线索连贯

| 不连贯类型 | 说明 | 示例 |
|------------|------|------|
| 线索断档 | 埋下的线索 / 支线写到一半无疾而终 | 第2章花大篇幅铺垫"神秘项链的来历"，此后全书再不提 |
| 情节硬切 | 事件之间"啪"地切换，无因果衔接 | "她拒绝了求婚"→（无任何过渡）"半年后她已在巴黎定居" |
| 伏笔回应突兀 | 伏笔回收缺少铺垫，像突然想起来 | 第1章随口提的"旧照片"，第10章突然成为认亲关键，中间无任何呼应 |
| 行动缺因果 | 人物做了一个重大决定，前面没有任何动机 / 诱因 | 女主突然"辞去工作远走他乡"，前文既无此念想也无触发事件 |

### 维度四：时空 / 场景连贯

| 不连贯类型 | 说明 | 示例 |
|------------|------|------|
| 时间跳跃无交代 | 时间跨度缺失，读者算不清"过了多久" | 第4章"春日踏青"，第5章"腊月里飘起雪"，中间无季节 / 月份过渡 |
| 空间转换突兀 | 地点切换让读者迷路，缺"怎么过去的" | "她在书房看书"→（无交代）"她站在码头等船"，读者不知何时到了码头 |
| 场景切换缺过渡 | 同一场景内镜头硬跳 | 正写客厅对话，下一句已在厨房做饭，中间无走动 / 时间交代 |
| 时空前后混淆 | 同一事物时间 / 地点前后不一致（非逻辑矛盾，更像忘了） | 前文"去年夏天相遇"，后文回忆成"前年冬天相遇"，无重述交代 |

### 维度五：视角 / 人称连贯

| 不连贯类型 | 说明 | 示例 |
|------------|------|------|
| 视角乱跳 | 限知视角内突然插入他人内心 | 本章固定女主视角，突然写"他其实早就知道真相"（越界） |
| 人称不一致 | 你 / 我 / 他混用 | 同一段叙事忽而"你看着他"，忽而"她看着他" |
| 叙述距离忽远忽近 | 同一章远距离概括与贴身特写无规律切换 | 前文"她度过了平常的一天"（远），下一句"她睫毛上挂着一滴将落未落的泪"（近），无节奏依据 |

### 维度六：心理动机连贯

| 不连贯类型 | 说明 | 示例 |
|------------|------|------|
| 行为动机缺失 | 人物行动缺内心依据，像被剧情推着走 | 她二话不说"替他顶了罪"，前文既无深爱也无亏欠铺垫 |
| 决策缺内心依据 | 关键抉择一笔带过，读者不知为何 | "她决定嫁给那个人"——无任何心理活动 / 权衡描写 |
| 性格→行为断裂 | 行为与人设性格不搭，但非矛盾（更像没写过渡） | 人设"内向慢热"，却在陌生聚会上一秒变社牛，无"被逼无奈 / 伪装"等过渡 |

### 章间连贯性专项扫描（相邻章衔接专项检测）

"情感 / 关系 / 情节 / 时空"四类不连贯，最容易在**相邻两章的衔接处**爆雷——前一章建立的连贯基线，后一章没接住。除逐维度比对外，**必须额外做一轮"相邻章连贯性扫描"**：

1. **依章节顺序逐对扫描**：对 (第1章,第2章)、(第2章,第3章)…(第N-1章,第N章) 每一对，核对前一章结尾建立的**连贯基线**与后一章开头是否平滑衔接。
2. **情感衔接检查**：前一章结尾的情绪状态（基调 / 余韵）是否在后一章开头被承接或合理反转（有触发事件）？还是直接"清零"或"跳变"？
3. **关系衔接检查**：前一章结尾的人物关系状态，后一章是否延续？关系若变化，中间是否有破冰 / 决裂事件交代？
4. **情节 / 线索衔接检查**：前一章结尾推进的线索 / 悬念，后一章是否接着走？有无"写到一半换话题"的断档？
5. **时空衔接检查**：后一章开头的时间 / 地点是否与前一章结尾连贯？跨度是否交代？
6. 发现不连贯即记入报告，标注「第X章 → 第X+1章」与断层点。

> 典型章间不连贯示例：
> - 第3章结尾"她终于鼓起勇气，决定向他表白"；第4章开头"她早已忘了那个人，安心过着独身生活"——情感与行动基线直接断裂，中间无任何打击 / 事件。
> - 第5章结尾"两人在雨中紧紧相拥，冰释前嫌"；第6章开头"她对他仍是戒备森森，寸步不让"——关系状态前后错位，像忘了衔接。
> - 第7章"他转身进了书房"；第8章"两人已在千里外的山庄里品茶"——空间硬切，读者不知怎么到的山庄。

## 三、连贯性修复主循环（可能需要多轮）

> **核心原则：先确认「连贯基线」，再动手补衔接。** 不连贯的"前文"通常才是已建立的基线（正确状态），修复的方向是**让后文平滑接上基线**——补过渡 / 补动机 / 补交代，而非推翻前文。
> **运行模式**：本阶段按第二节选定的 [A 自动] 或 [B 手动] 执行。自动模式走 3.2-A；手动模式走 3.2-B（每处都提供选项）。

### 3.1 锚点判定规则（自动模式的默认 / 手动模式的推荐基准）

| 场景 | 锚点（权威方） | 动作 |
|------|----------------|------|
| 后文与前文情感 / 关系 / 状态基线冲突或脱节 | **前文建立的连贯基线** | 在后文补过渡 / 交代，使后文平滑接上前文；不推翻前文既定内容 |
| 情节线索断档（伏笔 / 支线无下文） | **前文埋下的线索** | 在后文补呼应 / 回收，或补一笔"为何搁置"的交代；不删除前文铺垫 |
| 时空 / 场景硬切 | **前文结尾的时空状态** | 在后文开头补"时间过了多久 / 怎么到的某地 / 场景如何切换"的过渡句 |
| 行为 / 决策动机缺失 | **人物既定性格 / 人设 / 前文处境** | 补一段内心活动 / 诱因，让行为有据可依；不改成不符人设的行动 |
| 视角 / 人称乱跳 | **本章设定的主视角 / 人称** | 把越界部分收回主视角，或统一人称；不混用 |
| Timing 1 用户主动修改过设定 | **用户修改后的设定** | 以新设定为基线，给后文补衔接使其对齐新设定 |
| 前文本身才是断层根源（前文硬切 / 前文情绪无依据） | **后文更合理的连贯走向** | 回改前文，补前文的过渡 / 动机，使前后都连贯 |

> **女频侧重**：情感与关系（维度一、二）是女频读者最敏感的连贯点，修复时优先保证"情绪有铺垫、关系有过渡、CP 有节奏"。可用 phase7（情感浓度）、phase8（叙述手法）、phase9（CP 张力）的技巧补衔接——例如用「动作细节替代情绪标签」承接情绪，用「一个微小互动」承接关系进展。

### 3.2 修复执行（逐文件写回）

#### 3.2-A 自动模式（选 [A] 时）

对每一处选中的不连贯，按固定顺序处理：**定位 → 套用 3.1 规则确认锚点 → 补衔接 / 改硬切 → 单文件落地 → 记录**。不打断用户。

> **自动模式补衔接示例**：
> - 不连贯（第4章 甜蜜→崩溃无铺垫）：锚点=前文甜蜜基线 → 在第4章补一段"他那条消息弹出时，她脸上的笑一点点僵住"的触发 + 积累，让崩溃有据可依。
> - 不连贯（第5章→第6章 陌路→亲密无破冰）：锚点=第5章陌路基线 → 在第6章补一个"一次意外并肩作战，让两人对彼此改观"的破冰事件作为过渡。
> - 不连贯（第7章→第8章 书房→千里外山庄硬切）：锚点=第7章书房时空 → 在第8章开头补"当夜他派人护送她南下，三日后车轮停在一处依山傍水的山庄前"。
> - 不连贯（第7章 替他顶罪无动机）：锚点=人设 + 前文处境 → 补一段"她想起三年前那场雨里，是他还自己一条命"的内心回忆，动机落地。

#### 3.2-B 手动逐条模式（选 [B] 时）— 每处不连贯都提供选项

按不连贯编号从小到大，对**每一处**单独处理，**不合并、不批量默认**：

1. **呈现该不连贯**：用文字说明「编号 / 维度 / 位置（第X章 或 第X章→第X+1章）/ 断层表现 / 候选补法（来自下表）」。
2. **给出可选补法（AskUserQuestion）**：从该维度候选补法表取出 2~3 种方案，请用户**选一种作为基准**；额外提供 **[我自己指定补法]** 与 **[对剩余同类套用此方案]**（快捷项）。
3. **落地**：按用户所选方案补衔接，**仅写回涉及的那一个文件**；记录「编号、所选方案、涉及文件」。
4. 处理**下一处**（回到步骤 1），直到全部处理完。

**各维度候选补法表**（手动模式据此生成每处选项）：

| 不连贯类型 | 候选补法（供用户选一种作为基准） |
|------------|----------------------------------|
| 维度一·情感 | ① 在前文补触发 / 铺垫，让后文情绪有据可依（默认）② 在后文补余韵 / 缓冲，承接前文情绪 ③ 补一段内心活动交代情绪转折 ④ 我自己指定 |

> 手动模式示例（某处「第5章→第6章 陌路→亲密无破冰」）：

> 用户选「补破冰事件作过渡」→ 在第6章开头加一段两人被迫合作的破冰戏 → 仅写回第6章。

> 自动 / 手动共用的改写对齐细则：
> - 改**后文**：以后文为修复落点，补过渡 / 动机 / 交代，使其平滑接上前文基线；保持"展示而非讲述"、不破坏悬念钩子与人物弧光。
> - 改**前文**（仅当 3.1 判定前文本身是断层根源时）：补前文的过渡 / 动机，使前后都连贯。
> - 单文件落地：一次只写回一个文件。若断层横跨两章，只在需要修正的那一章修改。

### 3.3 多轮复检（连贯性可能多处相连）

连贯断层**往往一处带出一串**（情绪断了 → 关系也显得突兀）。因此：

1. 本轮所有选中不连贯修复完成后，**重新执行一次「一、连贯性检测」**，基于修改后的材料复扫。
2. 若复检仍发现不连贯（含修复引入的新断层）：
   - 生成**更新后的报告**，回到「二、呈现与用户选择」请用户决定下一轮是否继续。
   - 如此循环，直到**无不连贯剩余**或**用户选择跳过结束**。
3. 若复检无不连贯 → 进入第四步收尾。

## 四、修复后检查

对每一处修改完成后的材料执行：

1. **字数检查**（仅当正文章节被改写时）：
   `python scripts/check_chapter_wordcount.py <章节文件路径>`
   - 若删除后字数低于 2000 → 用 [content-expansion.md](../guides/content-expansion.md) 补充**新**内容，且新内容不得引入新断层。
2. **连贯性复核**：补的衔接是否与前后文自然融合，黄金法则（展示而非讲述 / 冲突驱动 / 悬念钩子）是否仍满足。
3. **无新断层检查**：确保补的过渡没有意外制造新的情绪 / 关系 / 时空断裂。
4. **不破坏结构**：章节结尾悬念钩子、大纲 `keyEvents` 覆盖度、人物弧光不被削弱。

## 六、避坑指南

### 不要过度修改（补衔接 ≠ 重写）
- 连贯性修复是"打补丁"——补一句过渡 / 一段内心 / 一个破冰事件即可，不要为消除一处小断层而重写整章。
- 保留角色标志性特征与必要承接（1~2 句足够），这是辨识度与连贯性的来源。

### 不要破坏悬念与弧光
- 章节结尾钩子即使与上文需补衔接，优先在后文开头补承接，而非删结尾钩子。
- 补情绪 / 关系过渡时，保留该章的冲突 / 转折力度（黄金法则二）。

### 不要引入新问题
- 补的过渡不能比原文差（抽象程度、细节密度）。
- 跨章节引用保持一致（角色姓名、地点名、设定名词不改错）。
- 改 JSON / 章节后，确认 `01-大纲.json` 与 `02-写作计划.json` 的章节 `title` / `chapterNumber` 仍对应。

### 尊重用户意图
- Timing 1 用户主动改过的设定即视为最终意图基线，补衔接以新设定为准，不擅自"还原"回旧设定。
- 任何无法判定锚点的不连贯，交回用户决策，不替用户拍板。

### 与第10阶段的协作
- 第10阶段先修「**矛盾**」（逻辑冲突），本阶段接着修「**不连贯**」（衔接断层）——推荐顺序：第10阶段（矛盾打底）→ 本阶段（连贯衔接）→ 第7/8/9 阶段（女频润色升华）。
- 若一处既是矛盾又是不连贯，优先按第10阶段逻辑判定"能否同真"定性质，再决定走哪一阶段。
